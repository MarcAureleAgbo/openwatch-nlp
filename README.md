# OpenWatch NLP

NLP classification prototype for detecting weak signals in public text content and classifying them into three risk levels: `red`, `orange`, and `green`.

## Business Problem

Organizations often need to monitor large volumes of public text such as forum posts, reviews, comments, or support discussions. The goal is not to automate final decisions, but to prioritize signals that may deserve human review.

This project demonstrates a simple and explainable NLP pipeline for weak-signal classification.

## Project Overview

OpenWatch NLP uses a synthetic dataset of short public-style texts. Each text is assigned to one of three labels:

- `red`: critical signal requiring priority review;
- `orange`: intermediate signal requiring monitoring;
- `green`: non-relevant or neutral content.

The first version uses a classic machine learning baseline:

- text cleaning;
- TF-IDF vectorization;
- multinomial Logistic Regression;
- train/test evaluation;
- confusion matrix;
- Streamlit interface for manual predictions.

## Dataset

The dataset is fully synthetic and generated by `src/data_generation.py`.

Main columns:

| Column | Description |
|---|---|
| `text_id` | Unique synthetic identifier |
| `text` | Raw synthetic text |
| `label` | Target class: `red`, `orange`, or `green` |
| `source_type` | Synthetic source category |
| `created_at` | Synthetic publication date |

This project does not use confidential, private, or scraped data.

## Technical Approach

The model uses `TfidfVectorizer` to convert text into numerical features. A `LogisticRegression` classifier then learns patterns associated with each risk level.

This is intentionally simple. For a portfolio project, a transparent baseline is easier to explain, debug, and defend than an unnecessarily complex model.

## How to Run Locally

Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Generate the synthetic dataset:

```bash
python -m src.data_generation
```

Train the model:

```bash
python -m src.train_model
```

Evaluate the model:

```bash
python -m src.evaluate_model
```

Run a single prediction:

```bash
python -m src.predict "A public post claims that administrator credentials were leaked."
```

Run the Streamlit app:

```bash
streamlit run app/streamlit_app.py
```

## Docker

Build the image:

```bash
docker build -t openwatch-nlp .
```

Run the app:

```bash
docker run -p 8501:8501 openwatch-nlp
```

## Results

The evaluation script writes:

- `reports/classification_report.txt`;
- `reports/metrics.json`;
- `reports/confusion_matrix.csv`;
- `reports/figures/confusion_matrix.png`.

Because the dataset is synthetic and pattern-based, performance may be high. The important point in this version is the end-to-end pipeline, not claiming production-grade generalization.

## Limitations

- The dataset is synthetic and simplified.
- The model may learn template-specific wording.
- There is no real-time ingestion.
- There is no human feedback loop.
- The model should not be used for operational decisions without real validation.
